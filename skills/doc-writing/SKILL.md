# ドキュメントライティング（本人コーパス駆動）

AI が書いた文章は、内容が正しくても「読者を説得しにいく語り」「論評」「対称的な網羅」で本人の文書から浮く。
このスキルは、本人の実文書から抽出した原則と型で文書を書くための規範を定める。

## 文体の核（最初に憶えること）

**言い切るのは構造（目的・定義・ルール・事実）、留保するのは人と未来（評価・推測・未決）。**

- 会議のゴールや運用ルールは「〜とする」「〜こと。」で言い切る
- 人への評価・自分の提案の位置づけ・自己評価は「たたき台」「〜と推測する」「結論めいた」で留保する
- 提案書ではこれが逆転の形をとる: Why 層（ユーザー価値・必要性）は強く言い切り、自分の利害は「正直瑣末な問題」と手放す。主張の強度と自我の強度を逆転させる

## ワークフロー

1. ジャンルを特定する（ガイドライン / 提案書 / 分析レポート / アジェンダ・会議メモ / 1on1 / PR本文 / README・運用ルール / 告知 / エッセイ）
2. `references/genres.md` の該当ジャンルの型を読む
3. 骨格チェック（下記）を通す。骨格が無い文書は文体を整えても直らない
4. 一般原則（下記）を当てながら書く
5. `references/voice.md` で語彙・記法・反面リストを照合する
6. 変更を報告する。使った型・削ったもの・判断に迷った箇所を列挙し、本人が検収できる形にする

既存文書のリライト時は、大筋（構成・結論・決定事項）を変えず声と構造だけ差し替える。
コーパスは `references/corpus.md` にある。型と実物がずれたら実物を正とし、このスキルを更新する。

## 骨格チェック（文体より先）

- 目的：文書・会議のゴールを冒頭一文で言い切る。達成判定できる動詞で書く（「終わらせ方を考えます」「〜を合意することです」）
- 決めること：番号付きで列挙。「決めないこと」「やらないこと」も理由付きで書く
- 判定ルール：振り分け・優先度・採否を扱うなら、判断より先に基準と用語定義を置く。既存の運用ルールがあるなら発明せず参照する

## 一般原則

1. **目的を冒頭一文で言い切る**。読者が1文目で「読むべきか」「何を持ち帰るか」を判定できる
2. **「なぜ」を独立させる**。目的の直後に「なぜこれが必要か」節を置く。意図が共有されるとルールを字義でなく目的で運用してもらえる
3. **長い文書には TL;DR**。「現状 → リスク → 提案」の3ブロックで圧縮する
4. **判断の前に定義**。「本MTGではこの定義に従った言葉を使用してください」と宣言してから番号付きで定義を置く。概念の混線を文書側で防ぐ
5. **対比の一行変換**。似た概念は「今やること／次にやること／いつかやること」のような写像1行で区別する。長文説明より記憶に残る
6. **たたき台宣言で改訂権を渡す**。「※以下はたたき台です」「自由に編集してください！」。決定でなくたたき台と位置づけると反発が減り、当事者性が残る
7. **読者の防御反応を先回りで解除する**。「特定の個人の評価を目的とするものではありません」「〜と言いたいのではなく〜」。センシティブな主張ほど誤読を本文側で潰す
8. **ロール別・定型反復**。読者ロールごとに節を割り、「目的：／参加者：／頻度：」のような同一テンプレを反復する。読者は自分の節だけ読めばよく、項目の欠落が「未決」として見える
9. **段階分けと移行条件**。施策は短期/中期/長期に分け、「効果が見えれば正式化します」と昇格条件を書く。反対されそうな案には数字付きのフォールバック（「暫定的（6ヶ月以内）」）を自分から用意する
10. **事実・数値で駆動する**。「議論や希望的観測を挟まず、数値(事実)を確認します」。定量/定性を分けて列挙し、数字なしの成果主張をしない。伝聞は伝聞と明記する
11. **ルールには理由と例外**。「〜を防ぎたいのでそれが満たせてればOK」。目的が書いてあれば読者が例外ケースで正しく曲げられる。よくない例/よい例はセリフ単位で添える
12. **やらないこと・限界・残論点を明記する**。先送りには再開条件を添える（「必要になったら再検討」）。無いときも「とくになし」と書く（空欄は書き忘れと区別がつかない）。提案書は「かけていないこと」を末尾で自己開示する
13. **正本を宣言し、詳細はリンクに委ねる**。「〜が正本」「このカタログが唯一の真実」を一箇所に。経緯はリンク一語で済ませ、本文は判断だけ持つ
14. **未確認は未確認のまま出す**。チェックボックスを塗りつぶさない。「公式ドキュメントに記載がなく未確認」。この正直さが文書全体の信頼を担保する
15. **収束の配管を敷いて閉じる**。ネクストアクションは担当者・順序・持ち帰り先まで。発散の場では「まとめは後日わたしが推進します」とオーナーシップを先に宣言する

## 書き分けの法則

- **経緯は PR に、現在形の理由は恒久文書に**。PR 本文では経緯を番号リストで厚く語り、README・ルール文書では「なぜ今こうか」だけを現在形で書く
- 恒久文書の規範文には、破ったとき何が起きるかを括弧の口語で即答する（「（聞かれる側が困る）」「（skip だとハッシュ不変で二度と走らない）」）

## 併用

- japanese-tech-writing: 文レベルの一般規範。常に併読する
- pr-description-style / pr-reviewer-context: 対象が PR のときの上書き層

## 更新方針

原則・型・声はすべて実文書からの観測であり、本人の文書は更新され続ける。
新しい本人手書き文書を読んだら答え合わせをし、ずれていたらこのスキル側を直す。
